# Figmaデザインのコーディングルール

Figmaデザインをコーディングする際のルールを定めます。

## Figma MCPの利用方法

### MCPツールの補足

- `get_meta_data`:
  - XML形式のメタデータが取得できる
  - ファイルに保存する必要がある場合は、`.xml`の拡張子で保存する

- `get_design_context`:
  - 「React + Tailwind CSS」のJSX形式のコードが取得できる
  - ファイルに保存する必要がある場合は、`.jsx`の拡張子で保存する
  - コンテンツの量が大きすぎる場合、`get_meta_data`と同じXML形式のメタデータが返ってくる場合がある

### Figmaのデザインを実装する場合のMCPの使用ルール

- `get_design_context` と `get_screenshot` は必ず使う
- `get_design_context` と `get_screenshot` のどちらかの実行結果が正しく得られなかった場合、処理を中断する
- `get_design_context`の中に変数が含まれていた場合、`get_variable_defs`も使う

## テキスト

Figmaのデザインに含まれているテキストは必ずそのまま実装に反映する。  
勝手にテキストを改変してはいけない。

## 画像

画像やイラストなどが使用されている要素はプレースホルダー画像を使用して実装する。
プレースホルダー画像は `https://placehold.co/` のサービスを利用する。

### プレースホルダーで必ず指定するもの

- デザインで指定された要素のサイズ(横幅x高さ)をプレースホルダー画像のサイズに指定する
- 背景色は「#808080」、前景色は「#ffffff」で必ず固定する
- テキストは入力してはいけない(サイズが表示されている方が確認しやすい)

## Webフォント

デザインでGoogle Fontsが使われている場合は、<head>タグ内にGoogle Fontsのリンクを追加する。  
`global.css`にimportしてはいけない。
